Technical Details
Client-friendly explanation.
How it works

Backend performs: OAuth code exchange → download eAadhaar XML → parse XML (Name/DOB/Address/Last-4) → persist last-4 only.

API Endpoints
  • GET /api/aadhaar/health — health check
  • GET /api/aadhaar/digilocker/authorize-url?state=... — generates OAuth authorize URL
  • POST /api/digilocker/mock-verify — mock verification (demo mode)
  • GET /api/aadhaar/verifications/recent — recent verification rows
Security & Compliance
  • Full Aadhaar number is never stored or returned.
  • Only last 4 digits extracted from XML are persisted.
  • Compliance notice is emphasized in UI.
Status

Live indicator updates from backend health. If backend is offline, UI falls back to mock output.
